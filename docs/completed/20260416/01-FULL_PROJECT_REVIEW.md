## 任务目标

对 `vldb-lancedb` 项目进行一次完整工程审查，重点覆盖核心入口、运行时、配置、服务层、FFI 暴露、数据引擎与示例客户端，识别会影响正确性、稳定性、性能、安全性或可维护性的离散问题，并形成可执行的审查结论。

## 执行步骤

1. 梳理仓库结构，识别核心模块、入口文件、构建脚本与示例代码。
2. 优先审查启动链路与配置加载流程，确认初始化、默认值与错误处理是否可靠。
3. 审查运行时、管理器、服务层与引擎层的调用关系，定位并发、资源生命周期与异常传播风险。
4. 审查 FFI、类型定义与对外接口，检查跨边界调用的内存安全、参数约束与返回语义。
5. 抽查示例客户端与文档约定，核对对外使用路径是否与实现一致。
6. 汇总问题并按优先级输出，仅保留作者明确会愿意修复的离散缺陷。

## 技术选型

- 使用 `git ls-files`、目录扫描与源码阅读梳理项目结构。
- 使用 CodeKit 工具做目录级和文本级定位，快速缩小高风险函数范围。
- 以静态审查为主，必要时辅以测试或命令验证结论。

## 验收标准

- 覆盖主要 Rust 源码文件及关键对外接口文件。
- 每个问题都具备明确位置、触发条件、影响说明与可操作性。
- 若未发现值得作者立即修复的问题，明确给出“项目当前审查范围内未发现离散缺陷”的结论。
